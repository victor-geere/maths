# CLAUDE.md — project/

One Markdown note per catalogue object in the circle-kernel track (track 1 — see root
[CLAUDE.md](../CLAUDE.md) §Two tracks). Each note applies the damp → symmetrise → kernel recipe
described in [victor/CLAUDE.md](../victor/CLAUDE.md) §Architecture to a different arithmetic
sequence, and is implemented numerically against the canonical `channels`/`sign_aware_helix`/
`quaternion_cone` pattern in `victor/prime-zeros.py`.

For per-phase task status and the open-work board, see [plan/CLAUDE.md](../plan/CLAUDE.md) and
`plan/project.md`. For the parallel structured research notes (phase folders, gate conditions,
deliverables), see [research/CLAUDE.md](../research/CLAUDE.md).

Rigour tags, the RH declaration bar, and math-rendering convention are defined once in root
[CLAUDE.md](../CLAUDE.md) and bind every note in this folder unchanged. `prime-sine-wave.md` is
the canonical template for a *proven* note (four theorems, honest §7 scope disclaimer, 10-digit
numerical match) — follow its structure for new catalogue notes.
